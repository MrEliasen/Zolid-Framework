Installing

Installing the Zolid Framework is very easy. You simply upload your files to your web host, and navigate your browser to where you uploaded the files, and the installer should open automatically, else navigate to it by appending "/install" (without the quotes) to the url. Example: www.domain.com/install

Deleting the install.php file from the "/ core / templates /" directory is recommended once the install is complete.

Adding Pages

Adding pages

Default pages are build up the folling way:

A) This part blocks users from viewing the file directly, eg. if the navigate to the file directly.

B) This part means that if the user is not logged in, redirect them to the frontpage/base url of the site. Of cause this is only required if you want the page to be for members only.

C) This is where your HTML, additional PHP or like goes. This is your page's content.

Remember when you add new pages to update your navigation as well in the "generateNavigation()" function in the file: "/ core / classes / template.class.php "

General Useage

Most of the classes are loaded together, and can be accessed with $this in eg. your template files. You can open any of the classes, and read through the documentation block at the top of each function to learn more about what each function do.

There are a couple of static classes as well, which can be accessed anywhere:

  • Notifications
  • Security
  • Cache

To give you an example of how to use a static class anywhere you wish, I'll show you how to use the notifications class:

You can set and show notifications using this class. The way to use this class is as follows:
Notifications::function_name( $variables );

Set new notifications to be displayed:
Set the notification to display to the user at the given position on your site.

$where (string) - the identifier for where to display the notification (see the showNotification function)
$body (string) - the notification text
$title (string) - the notification title (can be left blank)
$type (string) - can be one of the following: error, warning, success, info

Example: Notifications::setNotification( $where, $body, $title, $type );


Show notifications:
Show notifications which are set for the specific area you specify. If no notification is available, it will return nothing.

 

$where (string) - the identifier used to look for notification. Eg. if you set a notification with $where = login_1, you need to use this for fetching the notification here

Example: Notifications::showNotification( $where );
